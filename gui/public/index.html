Control Room

Run activity safely with preview, preflight checks, templates, scheduling, and live logs.

Default: dry-run safe mode Activity: commits Mode: dry-run Run: idle

How To Use

1. Pick token source

Use the .env token checkbox or paste a GitHub token manually in the token field.

2. Paste repository link

Paste your GitHub repository URL. Owner and Repo fields will auto-fill.

3. Choose activity

Select commits, issues, pull requests, reviews, or all. Start with dry-run first.

4. Set counts and dates

Adjust counts, pick random or range dates, and use calendar preview if needed.

5. Run pre-flight

Use Pre-flight to catch missing inputs before execution.

6. Run dry first

Dry-run shows what would happen without writing anything to GitHub or Git.

7. For live commits or PRs

This folder must be a real Git repository. If not, live commits and PRs will be blocked.

8. If Git repo is missing

Either clone the real repository into this folder or run: git init, git branch -M main, then add your remote.

9. Use live mode carefully

Switch to live mode only after dry-run looks correct. You will need to type YES to confirm.

10. Review output and history

Watch logs, stop a run if needed, and use Run History to reapply past settings.

Tokens are never saved to defaults/history.
